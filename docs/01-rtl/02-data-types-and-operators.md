# 1.2 数据类型与运算符

## 1. 四态逻辑与位宽

Verilog 信号在仿真中有 **0、1、X（未知）、Z（高阻）** 四态；综合工具通常将 RTL 映射为 **0/1** 二值逻辑。写 RTL 时应避免依赖 X 传播来“掩盖”未初始化（综合后行为未定义）。

**位宽规则（务必掌握）：**

- 表达式位宽由 **操作数中较宽者** 决定（有符号运算另见 `$signed`）。
- **赋值截断**：右值位宽大于左值时，高位被截断，可能无声丢失数据。
- **无符号 vs 有符号**：比较与移位行为不同，混用时常出 bug。

```verilog
wire [7:0]  a = 8'hFF;
wire [7:0]  b = 8'd1;
wire [8:0]  sum_ok = a + b;      // 9 位，无溢出
wire [7:0]  sum_bad = a + b;     // 截断为 8 位，溢出
```

**建议**：算术前先 **显式扩展** 到位宽。

```verilog
wire [15:0] prod = a * b;  // 若 a,b 为 8 位，先考虑是否需要 16 位中间结果
```

## 2. 常用数据类型

| 类型 | 用途 | 综合注意 |
|------|------|----------|
| `wire` / `logic` | 线网、连续赋值 | 多驱动需避免（三态总线除外） |
| `reg` / `logic` | `always` 内赋值 | 组合 always 用阻塞；时序用非阻塞 |
| `integer` | 仿真循环变量 | 一般不用于可综合 RTL |
| `parameter` / `localparam` | 常量 | 用于位宽、深度、状态编码 |
| `typedef struct/enum` (SV) | 结构化类型 | 推荐用于 FSM 状态（见 05 章） |

### 向量与定界

```verilog
logic [31:0] data;
logic [0:31] data_rev;  // 位序与 [31:0] 相反，切片时注意

assign byte0 = data[7:0];
assign msb   = data[31];
```

## 3. 运算符（可综合子集）

### 算术

`+` `-` `*` `/` `%` — 乘除法消耗资源大，高频路径慎用；常数除法可综合为移位/移位加。

### 按位

`&` `|` `^` `~` `{ }` 拼接 — 综合为组合逻辑，代价低。

### 逻辑与关系

`&&` `||` `!` `==` `!=` `<` `>` 等 — 结果为 1 位；注意 **逐位** `&` 与 **逻辑** `&&` 区别。

### 移位

`<<` `>>` 逻辑移位；`<<<` `>>>` 算术移位（与符号位相关）。常数移位综合为布线重排或少量逻辑。

### 缩减运算符

`&a` `|^a` 等 — 对向量归约，常用于 **parity、全零/全一检测**。

### 条件运算符

```verilog
assign sel_out = sel ? a : b;  // 综合为多路选择器
```

## 4. 赋值语句类型

| 语句 | 出现位置 | 用途 |
|------|----------|------|
| 连续赋值 `assign` | 模块级 | 组合逻辑、线网驱动 |
| 阻塞 `=` | `always` 组合 | 顺序执行，模拟硬件并发时需小心 |
| 非阻塞 `<=` | `always` 时序 | 模拟 NBA 调度，对应寄存器更新 |

**铁律**：时序逻辑 **只用** `<=`；组合 `always` **只用** `=`（见下一章）。

## 5. 参数与 localparam

```verilog
parameter FIFO_DEPTH = 16;
localparam ADDR_W = $clog2(FIFO_DEPTH);

logic [ADDR_W-1:0] ptr;
```

- `localparam` 不对外暴露，避免层次化误改。
- `$clog2(N)` 在 elaboration 求值，用于地址位宽。

## 6. 小结

- 在位宽边界 **显式扩展/截断**，并做仿真断言检查溢出。
- 区分按位、逻辑、算术移位与有符号属性。
- 用 `parameter` 管理可配置位宽，避免魔数。
